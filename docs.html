Introduction

Layer 10 Security MCP Gateway is an enterprise-grade Model Context Protocol gateway designed specifically for security operations. It enables AI agents to securely interact with security tools while maintaining compliance, audit trails, and risk-aware access control.

What is MCP?

The Model Context Protocol (MCP) is a standardized protocol for AI systems to interact with external tools and data sources. Layer 10 extends MCP with enterprise security features.

Key Features

  • Task-Based Access Control (TBAC) - Context-aware authorization beyond traditional RBAC
  • Deep Payload Inspection - Detect prompt injection and command injection attacks
  • Real-time Risk Scoring - ML-powered assessment of every request
  • Forensic Audit Logging - SIEM-integrated, immutable audit trails
  • Enterprise IdP Integration - Okta, Entra ID, Google Workspace

Quick Start

Get up and running with Layer 10 Gateway in minutes.

Prerequisites

  • Docker 20.10+ or Kubernetes 1.24+
  • Node.js 18+ (for CLI tools)
  • An OAuth 2.1 compatible identity provider

Installation

Docker 
# Pull the Layer 10 Gateway image
docker pull layer10security/mcp-gateway:latest

# Run with default configuration
docker run -d \
  --name layer10-gateway \
  -p 8443:8443 \
  -e OAUTH_PROVIDER_URL=https://your-idp.com \
  -e OAUTH_CLIENT_ID=your-client-id \
  layer10security/mcp-gateway:latest
Kubernetes 
# Apply the Helm chart
helm repo add layer10 https://charts.layer10security.io
helm install layer10-gateway layer10/mcp-gateway \
  --set oauth.providerUrl=https://your-idp.com \
  --set oauth.clientId=your-client-id

Configuration

Configure the gateway using environment variables or a YAML configuration file.

config.yaml 
gateway:
  port: 8443
  tls:
    enabled: true
    certFile: /etc/layer10/tls.crt
    keyFile: /etc/layer10/tls.key

authentication:
  provider: oauth2
  oauth2:
    issuerUrl: https://your-idp.com
    clientId: your-client-id
    scopes:
      - openid
      - profile
      - email

authorization:
  mode: tbac
  policies:
    - name: security-scan
      tools: [nmap, nuclei, trivy]
      requiredScopes: [security:scan]

audit:
  enabled: true
  destination: splunk
  splunk:
    hecUrl: https://splunk.example.com:8088
    hecToken: ${SPLUNK_HEC_TOKEN}

riskScoring:
  enabled: true
  threshold: 0.7
  actions:
    high: block
    medium: log
    low: allow

Architecture

The Layer 10 Gateway consists of several interconnected components designed for security and scalability.

Components

  • API Gateway Layer - Handles authentication, rate limiting, and TLS termination
  • MCP Router - Session-aware routing to MCP servers
  • Policy Engine - Evaluates TBAC policies for each request
  • Risk Scoring Engine - ML-based threat assessment
  • Audit Logger - Immutable logging with SIEM integration

Authentication

Layer 10 Gateway supports OAuth 2.1 and mTLS for secure authentication.

Supported Identity Providers

  • Microsoft Entra ID (Azure AD)
  • Okta
  • Google Workspace
  • Keycloak
  • Any OAuth 2.1 / OIDC compliant provider

Security Note

Always use PKCE (Proof Key for Code Exchange) when configuring OAuth 2.1 clients. Layer 10 Gateway enforces PKCE by default.

Task-Based Access Control

TBAC goes beyond traditional RBAC by considering the context of each request, not just the user's role.

How TBAC Works

Instead of granting blanket permissions based on roles, TBAC evaluates:

  • Task Context - What is the AI agent trying to accomplish?
  • Tool Scope - Which specific tools are needed?
  • Data Sensitivity - What data will be accessed?
  • Risk Level - What is the assessed risk of this operation?
TBAC Policy Example 
policies:
  - name: vulnerability-scan
    description: Allow vulnerability scanning on approved assets
    conditions:
      - task.type == "vulnerability_scan"
      - target.asset_group in user.authorized_assets
      - risk.score < 0.8
    tools:
      - nmap
      - nuclei
      - trivy
    actions:
      - allow
      - audit

API Reference

The Layer 10 Gateway exposes a RESTful API for management and a WebSocket/SSE interface for MCP communication.

Base URL

https://gateway.layer10security.io/api/v1

Authentication

All API requests require a Bearer token obtained via OAuth 2.1 flow.

Authorization: Bearer <access_token>
